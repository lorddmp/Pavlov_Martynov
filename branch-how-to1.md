В системе Git есть команды для создания ветки, переключения на неё и отправки ветки на сервер. 

Создание ветки
Команда git branch <имя_ветки> создаёт новую ветку, оставив разработчика в прежней. 

Чтобы создать ветку и сразу переключиться на неё, можно использовать команду git checkout -b <имя_ветки>. Это сокращение для двух команд: git branch <имя_ветки> (создание ветки) и git checkout <имя_ветки> (переключение на неё). 

Если нужно создать ветку от определённого коммита или другой ветки, используют следующие команды: 

от определённого коммита: git checkout -b <имя_ветки> <хеш_коммита>;
от другой ветки: git checkout -b <имя_ветки> origin/develop.

Важно: новая ветка будет создана на основе той, которая была активна в момент создания, — она будет содержать все коммиты и файлы из «родительской» ветки.

Переключение
Команда git checkout <имя_ветки> переключает на другую ветку. 

Некоторые варианты использования:
переключение на существующую ветку: git checkout <имя_ветки>;
переключение на удалённую ветку: git checkout origin/<имя_ветки>.
 
Важно: перед переключением между ветками нужно убедиться, что все изменения сохранены (закоммичены). В противном случае Git не позволит переключиться или предложит сохранить изменения с помощью git stash. 

Если переключиться на другую ветку, не сделав предварительно коммит, все незакоммиченные изменения будут перенесены в ту ветку, на которую переключились.
